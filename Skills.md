# Skills.md — TradingView AI Analytics 분석 규칙

> 이 문서는 TradingView AI Analytics Dashboard의 데이터 분석 규칙과 생성 지침을 정의합니다.
> LLM 에이전트와 프론트엔드 시각화 엔진이 이 규칙을 기반으로 동작합니다.

---

## 1. 투자 데이터 분석 기준

### 1.1 엔티티 추출 규칙

사용자 자연어 입력에서 다음 엔티티를 추출합니다:

| 엔티티 | 설명 | 추출 예시 |
|--------|------|-----------|
| `symbol` | 종목/지수 심볼 | "삼성전자" → `005930.KS`, "NVDA" → `NVDA` |
| `period` | 분석 기간 | "최근 1달" → `1M`, "3개월" → `3M`, "올해" → `YTD` |
| `indicator` | 기술적 지표 | "RSI", "MACD", "볼린저 밴드" |
| `market` | 시장 구분 | "코스피" → `KR:KOSPI`, "나스닥" → `US:NASDAQ` |
| `action` | 요청 유형 | "차트 보여줘" → `chart`, "분석해줘" → `analysis` |

### 1.2 시장 구분 기준

| 시장 코드 | 통화 | 거래 시간 (현지) | 색상 규칙 |
|-----------|------|------------------|-----------|
| `KR` | KRW | 09:00~15:30 KST | 상승=빨강, 하락=파랑 |
| `US` | USD | 09:30~16:00 ET | 상승=초록, 하락=빨강 |
| `JP` | JPY | 09:00~15:00 JST | 상승=빨강, 하락=파랑 |
| `HK` | HKD | 09:30~16:00 HKT | 상승=빨강, 하락=초록 |
| `EU` | EUR | 09:00~17:30 CET | 상승=초록, 하락=빨강 |

### 1.3 기간 매핑

| 입력 키워드 | 코드 | 캔들 간격 | 데이터 포인트 |
|-------------|------|-----------|---------------|
| 오늘, 당일 | `1D` | 5분 | 78 |
| 1주, 이번주 | `1W` | 15분 | 130 |
| 1달, 한 달 | `1M` | 1시간 | 160 |
| 3개월 | `3M` | 일봉 | 63 |
| 6개월 | `6M` | 일봉 | 126 |
| 1년 | `1Y` | 일봉 | 252 |
| 올해 | `YTD` | 일봉 | 가변 |

## 2. 투자 지표 계산 규칙

### 2.1 기술적 지표 정의

#### RSI (Relative Strength Index)
- **공식**: RSI = 100 - (100 / (1 + RS)), RS = 평균 상승폭 / 평균 하락폭
- **기본 파라미터**: 기간 = 14
- **해석 기준**:
  - RSI ≥ 70: 과매수 (OVERBOUGHT) → 매도 신호
  - RSI ≤ 30: 과매도 (OVERSOLD) → 매수 신호
  - 30 < RSI < 70: 중립 → 홀드

#### MACD (Moving Average Convergence Divergence)
- **공식**: MACD Line = EMA(12) - EMA(26), Signal Line = EMA(MACD, 9)
- **기본 파라미터**: 단기 = 12, 장기 = 26, 시그널 = 9
- **해석 기준**:
  - MACD > Signal: 골든크로스 → 매수 신호
  - MACD < Signal: 데드크로스 → 매도 신호
  - |MACD - Signal| < 임계값: 중립

#### Bollinger Bands
- **공식**: 중심선 = SMA(20), 상단 = 중심선 + 2σ, 하단 = 중심선 - 2σ
- **기본 파라미터**: 기간 = 20, 표준편차 배수 = 2
- **해석 기준**:
  - 가격 > 상단밴드: 과매수 → 매도 신호
  - 가격 < 하단밴드: 과매도 → 매수 신호
  - 밴드폭 수축: 변동성 축소 → 돌파 임박 경고

#### Stochastic Oscillator
- **공식**: %K = (현재가 - 최저가) / (최고가 - 최저가) × 100
- **기본 파라미터**: %K = 14, %D = 3
- **해석 기준**:
  - %K ≥ 80: 과매수 → 매도 신호
  - %K ≤ 20: 과매도 → 매수 신호

### 2.2 투자 점수 환산 공식

각 지표의 신호를 0~100 단일 점수로 환산합니다:

```
총점 = Σ (지표별 점수 × 가중치) / Σ 가중치

지표별 점수:
  - 강한 매수 신호: 90~100
  - 약한 매수 신호: 70~89
  - 중립:          40~69
  - 약한 매도 신호: 20~39
  - 강한 매도 신호: 0~19

기본 가중치:
  - RSI:        0.25
  - MACD:       0.25
  - 볼린저밴드:  0.20
  - 거래량 추세: 0.15
  - 이동평균:    0.15
```

### 2.3 점수 → 신호 변환

| 점수 범위 | 신호 | 라벨 | 색상 |
|-----------|------|------|------|
| 80~100 | `strong_buy` | 적극 매수 | `#16a34a` |
| 60~79 | `buy` | 매수 | `#22c55e` |
| 40~59 | `hold` | 관망 | `#eab308` |
| 20~39 | `sell` | 매도 | `#f97316` |
| 0~19 | `strong_sell` | 적극 매도 | `#ef4444` |

## 3. 시각화 선택 기준

사용자 요청의 데이터 성격에 따라 최적의 차트 타입을 자동으로 선택합니다:

### 3.1 데이터 유형 → 차트 매핑

| 데이터 유형 | 차트 타입 | 위젯 크기 | 조건 |
|-------------|-----------|-----------|------|
| 시계열 가격 (OHLCV) | `candlestick` | 6×4 | 개별 종목 가격 조회 시 |
| 시계열 추세 | `line` (area) | 6×3 | 지수, 추세선, 비교 분석 시 |
| 비중/구성 | `donut` | 4×3 | 포트폴리오, 섹터, 구성비 |
| 상관관계 | `heatmap` | 6×4 | 2개 이상 종목 상관분석 시 |
| 분포/산점도 | `scatter` | 4×4 | 리스크-수익 분석 시 |
| 단일 수치 | `scorecard` | 4×3 | 지표 점수, 매매 신호 |
| 텍스트 요약 | `text` | 4×3 | 브리핑, 뉴스 요약, 인사이트 |
| 복합 비교 | `bar` | 6×3 | 여러 종목 수익률 비교 시 |

### 3.2 자동 선택 규칙 (Decision Tree)

```
입력 분석:
├── 종목명 + "차트/캔들/주가" → candlestick
├── "추세/트렌드/라인" + 종목명 → line
├── "포트폴리오/비중/구성/도넛" → donut
├── "히트맵/상관관계/상관" → heatmap
├── "비교/수익률" + 복수 종목 → bar
├── 지표명 (RSI/MACD/볼린저...) → scorecard
├── "브리핑/요약/뉴스/인사이트" → text
├── "점수/스코어/지표" → scorecard
└── 기타 (종목명만) → candlestick (기본값)
```

### 3.3 차트 스타일 규칙

| 속성 | 규칙 |
|------|------|
| 한국 시장 색상 | 상승 = `#dc2626` (빨강), 하락 = `#2563eb` (파랑) |
| 미국 시장 색상 | 상승 = `#22c55e` (초록), 하락 = `#ef4444` (빨강) |
| 배경 | 투명 (컨테이너 배경 활용) |
| 그리드 | `rgba(20,20,19,0.06)` — 최소한의 가이드라인 |
| 폰트 | Sofia Sans, 11px, `#6B6966` |
| 워터마크 | 비활성화 (숨김 처리) |

## 4. 리포트 구성 흐름

대시보드 분석 결과를 구조화된 리포트로 변환하는 흐름:

### 4.1 리포트 구조

```
리포트
├── 1. 헤더
│   ├── 생성 일시
│   ├── 대상 종목/시장
│   └── 분석 기간
├── 2. 시장 개요
│   ├── 글로벌 지수 요약 (등락률 테이블)
│   └── 환율 동향
├── 3. 종목 분석
│   ├── 가격 차트 스냅샷
│   ├── 기술적 지표 요약 카드
│   │   ├── RSI 점수 + 해석
│   │   ├── MACD 상태 + 해석
│   │   └── 볼린저 밴드 위치 + 해석
│   └── 종합 투자 점수 (0~100)
├── 4. AI 인사이트
│   ├── 3줄 요약 브리핑
│   ├── 신호 충돌 중재 결과 (있을 경우)
│   └── 과거 적중률 참조
└── 5. 결론
    ├── 최종 투자 의견 (매수/매도/관망)
    └── 주요 리스크 요인
```

### 4.2 리포트 생성 트리거

| 트리거 | 설명 |
|--------|------|
| GNB "리포트" 탭 클릭 | 현재 대시보드 위젯 기반 자동 생성 |
| 프롬프트: "리포트 생성" | 현재 분석 상태를 리포트로 변환 |
| 스냅샷 공유 버튼 | 리포트 URL 생성 + 클립보드 복사 |

## 5. 투자 인사이트 생성 규칙

### 5.1 3줄 브리핑 템플릿

LLM이 시장 인사이트를 생성할 때 반드시 다음 구조를 따릅니다:

```
1번째 줄: [현상] — 가격/지표의 현재 상태를 사실 기반으로 서술
2번째 줄: [원인] — 해당 변동의 주요 원인 (매크로/실적/수급)
3번째 줄: [전망] — 단기 전망과 주의할 포인트
```

**예시:**
```
• NVDA 전일 대비 +3.45% 상승, 52주 신고가 근접 ($125.80)
• AI 칩 수요 확대에 따른 2분기 실적 가이던스 상향이 주요 동인
• RSI 72 과매수 구간 진입, 단기 차익실현 매물 출회 가능성 주의
```

### 5.2 인사이트 금지 규칙

- 뉴스 원문을 그대로 복사하지 않음
- "~할 것으로 보입니다" 등 모호한 예측 표현 최소화
- 수치 근거 없는 방향성 주장 금지
- 투자 권유/추천 문구 절대 불포함 (면책 필수)

### 5.3 충돌 중재 규칙 (Conflict Watchdog)

지표 간 신호가 상충할 때 적용하는 중재 알고리즘:

```
충돌 감지 조건:
  IF 지표A.signal ≠ 지표B.signal THEN conflict = true

중재 프로세스:
  1. 각 지표의 과거 적중률 조회 (최근 100거래일 기준)
  2. 적중률 기반 가중치 부여:
     보정 신호 = argmax(Σ 지표i.signal × 지표i.accuracy)
  3. 신뢰도 계산:
     신뢰도 = max(적중률) / mean(적중률) × 100

출력 형식:
  {
    "conflict": true,
    "indicators": ["RSI 과매수 (72)", "MACD 골든크로스"],
    "accuracy": 75,          // 과거 적중률 %
    "corrected": "매수 유지", // 보정된 최종 판단
    "reasoning": "과거 유사 패턴에서 MACD 기준 73% 확률로 상승"
  }
```

### 5.4 면책 조항

모든 리포트 및 인사이트 하단에 자동 삽입:

> 본 분석은 AI 기반 자동 생성 결과이며, 투자 자문이 아닙니다. 투자 판단의 책임은 투자자 본인에게 있습니다.

## 6. 위젯 타입 정의

프론트엔드에서 지원하는 위젯 타입과 렌더링 규칙:

| type | chartType | 렌더러 | 데이터 소스 |
|------|-----------|--------|-------------|
| `chart` | `candlestick` | LightweightCharts v5 `CandlestickSeries` | OHLCV API |
| `chart` | `line` | LightweightCharts v5 `AreaSeries` | 시계열 API |
| `chart` | `donut` | SVG 기반 커스텀 렌더러 | LLM 생성 |
| `chart` | `heatmap` | HTML Table + 색상 그라데이션 | 상관계수 API |
| `chart` | `bar` | SVG 기반 바 차트 | 비교 데이터 |
| `scorecard` | — | SVG 게이지 + 신호 라벨 | 지표 계산 |
| `text` | — | HTML 리스트 렌더러 | LLM 브리핑 |
