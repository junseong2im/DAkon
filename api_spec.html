0. 공통 사항

Base URL

Production: https://api.deepsr.com/v1
Development: http://localhost:8000/v1

인증 방식

Authorization: Bearer <JWT_TOKEN>

커스텀 헤더 (BYOK 키 전달 시):
X-LLM-Key: sk-xxxxx
X-Market-Key: xxxxx

공통 응답 형식

{
  "status": "success" | "error",
  "data": { ... },
  "message": "설명 메시지 (에러 시)",
  "timestamp": "2026-05-03T14:00:00+09:00"
}

에러 코드

HTTP의미설명
200OK정상 응답
400Bad Request잘못된 파라미터
401Unauthorized인증 실패 / API 키 누락
404Not Found리소스 없음
429Too Many RequestsAPI 호출 한도 초과
500Internal Server Error서버 내부 오류

1. 시장 데이터 API

1.1 OHLCV (캔들스틱 데이터)

프론트 사용처: Charts.createCandlestick(), Widget.renderChart()

GET /market/ohlcv
파라미터타입필수설명예시
symbolstring필수종목 심볼005930, AAPL
marketstring필수시장 코드KR, US
intervalstring선택캔들 간격5m, 1h, 1d
countnumber선택데이터 포인트 수60 (기본)
fromstring선택시작일 (ISO 8601)2026-01-01
tostring선택종료일 (ISO 8601)2026-05-03

Response

{
  "status": "success",
  "data": {
    "symbol": "005930",
    "market": "KR",
    "name": "삼성전자",
    "currency": "KRW",
    "candles": [
      {
        "time": "2026-04-01",
        "open": 72000,
        "high": 73500,
        "low": 71800,
        "close": 73200,
        "volume": 12500000
      }
    ]
  }
}
프론트 매핑: data.candles → Charts.createCandlestick(id, candles)
TradingView Lightweight Charts v5 포맷. time은 YYYY-MM-DD 문자열.

프론트 사용처: Report.generateInsights(), Widget.renderTextBriefing()

  "market": "KR",
  "topic": "market",

4. WebSocket (실시간)

WS /ws/realtime
// Subscribe
{ "action": "subscribe", "channels": ["kospi"], "symbols": ["005930"] }

// Server Push (tick)
{
  "type": "tick",
  "symbol": "005930",
  "data": {
    "time": "2026-05-03T14:30:00+09:00",
    "price": 61250,
    "change": 50,
    "changePct": 0.08,
    "volume": 125000
  }
}

// Connection Status
{ "type": "status", "status": "connected" }
{ "type": "status", "status": "disconnected", "reason": "heartbeat_timeout" }
프론트 매핑:
Charts.appendPoint(id, { time, value }) → 실시간 차트 업데이트
Store.setWsStatus(status) → GNB 연결 상태 배지

📌 참고: API 키는 BYOK 정책상 서버에 저장하지 않습니다. 프론트엔드 localStorage에만 암호화 저장됩니다.
이 문서는 프론트엔드 팀이 작성했으며, 백엔드 팀은 위 스펙에 맞춰 API를 구현해 주세요.
